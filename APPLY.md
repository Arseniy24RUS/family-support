# Применение патча

## Автоматический способ

Распакуйте архив в отдельную директорию и выполните:

```bash
node apply-platform-patch.mjs /абсолютный/путь/к/family-support
```

Установщик проверит структуру репозитория и контрольные суммы основных файлов, создаст резервную копию рядом с целевым репозиторием, скопирует патч и выполнит проверку проекта и новые модульные тесты. При неуспешной проверке изменённые файлы будут восстановлены автоматически.

Опция `--force` разрешает установку поверх изменённых `README.md`, `site/app.js` и `site/styles.css`; перед её использованием следует сохранить собственные изменения в Git. Опция `--no-check` отключает автоматический прогон, но после неё проверки необходимо выполнить вручную.

## Ручной способ

Скопируйте содержимое архива поверх корня репозитория, исключив `APPLY.md`, `apply-platform-patch.mjs` и служебный manifest архива. Не удаляйте `site/data/`, `site/assets/`, `site/vendor/`, `scripts/adapters/`, `scripts/update.mjs` и `.github/workflows/`.

После установки:

```bash
npm run check
npm test
npm run test:e2e
npm run serve
```

Проверьте вручную четыре маршрута: `/index.html`, `/situations.html`, `/compare.html` и `/methodology.html`.
