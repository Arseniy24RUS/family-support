# Навигатор мер поддержки семей с детьми

Готовый статический проект Института социальной демографии ФНИСЦ РАН. Интерфейс показывает краткие сведения о федеральных и региональных мерах поддержки, позволяет искать и фильтровать карточки и переводит пользователя на страницу внешнего источника с полным описанием.

В проекте нет системы аккаунтов, формы входа и отдельной «версии для слабовидящих». При этом сохранены обычные механизмы доступной HTML-разметки: клавиатурный фокус, подписи элементов управления, семантические заголовки и ссылка перехода к основному содержанию.

## Состав проекта

```text
site/                         готовый статический сайт
  index.html                  разметка
  styles.css                  адаптивное оформление
  app.js                      поиск, фильтры, каталог и выбор региона
  assets/                     логотипы, hero-фотография и favicon
  vendor/                     локальная сборка Lucide и её лицензия
  data/                       снимок каталога, список и GeoJSON регионов
scripts/                      парсер и служебные сценарии
  adapters/sovetmam.mjs       Playwright-адаптер «Шпаргалки для родителей»
  update.mjs                  обновление и запись проверенного снимка
  check-project.mjs           проверка целостности готового проекта
.github/workflows/            автоматическое обновление и GitHub Pages
tests/                        модульные тесты разбора карточек
e2e/                          браузерные проверки desktop и mobile
playwright.config.mjs         конфигурация Chromium и локального сервера
```

## Быстрый локальный запуск

Требуется Node.js 22 или новее.

```bash
npm ci
npm run check
npm test
npx playwright install --with-deps chromium
npm run test:e2e
npm run serve
```

После запуска сайт доступен по адресу `http://localhost:8080`.

Открывать `site/index.html` непосредственно через `file://` не следует: браузер ограничит загрузку JSON-файлов. Используйте встроенный локальный сервер.

## Первичная публикация на GitHub Pages

1. Создайте пустой репозиторий и загрузите в него всё содержимое архива, сохранив структуру каталогов.
2. Основную ветку назовите `main`.
3. В разделе `Settings → Pages → Build and deployment` выберите `GitHub Actions`.
4. Первый push проверяет и публикует включённый в проект рабочий снимок без обращения к внешнему каталогу.
5. Для немедленного обновления откройте `Actions → Проверка, обновление каталога и публикация → Run workflow` и оставьте включённым параметр получения свежих данных.

Плановый запуск установлен на первое число каждого месяца, 03:17 по московскому времени (00:17 UTC). После успешного парсинга новый снимок записывается в `site/data/` и публикуется. При ошибке проверки развёртывание прекращается, поэтому уже опубликованная версия сайта не заменяется повреждёнными данными.

## Обновление каталога вручную

```bash
npm ci
npx playwright install --with-deps chromium
npm test
npm run update
npm run check
npm run test:e2e
npm run serve
```

В проект включён проверенный рабочий снимок внешнего каталога. Команда обновления открывает пользовательскую страницу в настоящем Chromium, сверяет сообщаемое страницей число мер и видимые DOM-карточки, затем извлекает полный структурированный набор из Next.js Flight payload. Если формат структурированного набора изменится, используется DOM-fallback с последовательным раскрытием «Показать ещё». Пустой или неполный результат не перезаписывает предыдущий снимок.

Управляемые параметры парсера:

```text
HEADLESS=0                         запуск Chromium с окном
PARSER_TIMEOUT_MS=30000            timeout браузерных операций
SOVETMAM_URL=https://.../catalog   адрес проверяемого каталога
DEBUG=1 или TRACE=1                trace, screenshot и расширенная диагностика
CHROMIUM_EXECUTABLE_PATH=/path     нестандартный Chromium для закрытой среды
PLAYWRIGHT_PROXY=http://...        явный HTTPS-прокси
PLAYWRIGHT_IGNORE_HTTPS_ERRORS=1   только для доверенного перехватывающего прокси
```

Диагностика сохраняется в `debug/`; эта директория, отчёты Playwright, trace, видео и screenshots исключены из Git.

## End-to-end проверки

```bash
npm run test:e2e
npm run test:e2e:headed
npm run test:e2e:ui
```

Основной прогон выполняется в Chromium для desktop 1672×941 и mobile 390×844. Он проверяет локальные ресурсы и логотипы, header, интерактивную карту из 89 регионов, подсказки и выбор региона, фактическую статистику, поиск с нормализацией «е/ё», URL-состояние, региональный диалог, категории и уровни, постраничное раскрытие, внешние ссылки, клавиатурную доступность и отсутствие горизонтального переполнения. Сетевая выборка карточек источника включается параметром `CHECK_EXTERNAL_LINKS=1`; обычный push не зависит от доступности внешнего сайта, а ежемесячное и ручное обновление в GitHub Actions выполняет эту проверку обязательно.

## Российский self-hosted runner

Если `app.sovetmam.ru` ограничивает обращения из зарубежных сетей GitHub, зарегистрируйте runner на машине Института и замените в workflow:

```yaml
runs-on: ubuntu-latest
```

на:

```yaml
runs-on: [self-hosted, linux, x64]
```

Сервер в этой схеме выполняет только периодическую сборку. Сам сайт остаётся статическим и не требует постоянно работающего приложения, базы данных или открытого входящего порта.

## Данные и переходы

Локально сохраняются только поля, необходимые витрине: идентификатор, название, уровень, регион, категория, краткая аннотация, характер поддержки, источник и URL карточки. Полные условия, перечни документов и подача заявления остаются на внешнем ресурсе.

При выборе субъекта на интерактивной карте или в списке каталог показывает федеральные меры вместе с мерами выбранного региона. Подсказка карты сообщает число региональных и федеральных мер. GeoJSON и базовый список содержат 89 субъектов; фактическое наличие карточек определяется текущим снимком источника.

## Внешние компоненты

Пиктограммы поставляются локально в `site/vendor/lucide.min.js`. Текст лицензии Lucide находится в `site/vendor/LUCIDE-LICENSE.txt`. Сайт не загружает библиотеку иконок с внешнего CDN.
