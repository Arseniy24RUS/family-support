# Платформа мер поддержки семей с детьми

Статическая информационная платформа Института социальной демографии ФНИСЦ РАН и Совета матерей. Основной раздел сохраняет полный федеральный и региональный каталог, поиск, фильтры, карту и подробные карточки. Дополнительные самостоятельные страницы выполняют предварительный подбор по жизненной ситуации, межрегиональное сравнение структуры каталога и раскрывают методологию проекта.

Платформа не требует регистрации и не создаёт серверный профиль семьи. Ответы анкеты подбора обрабатываются в браузере. Результаты носят справочный характер и не заменяют решение уполномоченного органа.

## Пользовательские страницы

| Маршрут | Назначение |
|---|---|
| `site/index.html` | Существующий полный каталог мер поддержки |
| `site/situations.html` | Объяснимый предварительный подбор по жизненной ситуации |
| `site/compare.html` | Межрегиональное сравнение структуры региональных записей |
| `site/methodology.html` | Данные, алгоритмы, ограничения, приватность и исправления |

Новые инструменты намеренно реализованы отдельными страницами. Основной header каталога не используется как навигация по модулям; ссылки пока размещены в footer и могут быть перенесены в header после пользовательской апробации.

## Что сохранено в каталоге

Патч не меняет источник данных, парсер, процедуру обновления, подробные шарды, GeoJSON, assets и локальную сборку Lucide. Каталог остаётся главной страницей и сохраняет выбор региона, категорий и уровня, поиск, карту, статистику, постраничное раскрытие и диалоги подробных карточек.

К каталогу добавлены:

- устойчивый URL карточки через параметр `measure`;
- локальное избранное без аккаунта;
- копирование ссылки и сообщение о неточности;
- явная маркировка наличия точной официальной ссылки;
- осторожная индикация предполагаемого типа поставщика;
- пояснение, что отсутствие региональной записи означает пробел текущего источника, а не отсутствие поддержки;
- переименование редакционного блока в «Ключевые федеральные меры» без заявления об измеренной популярности.

## Подбор по жизненной ситуации

Подбор использует детерминированные правила из `site/lib/life-situation-engine.js`. После выбора региона и ситуации система оставляет федеральные карточки и карточки выбранного субъекта, сопоставляет текст с тематическими словарями, учитывает отмеченные обстоятельства и дополнительные слова, затем показывает причины включения каждой меры и критерии, которые требуется проверить.

Результаты группируются как наиболее релевантные по тексту, требующие проверки условий и связанные меры. Эти группы характеризуют тематическое совпадение, а не вероятность назначения. В публичную ссылку входят только регион и тип ситуации; уточняющие признаки и свободный текст не записываются в URL.

## Межрегиональное сравнение

Страница `site/compare.html` сопоставляет от двух до четырёх субъектов. В показатели региона включаются только записи с уровнем `regional`; федеральные карточки показываются отдельно как общий фон. Модуль рассчитывает число записей, тематическую широту, крупнейшую категорию, распределение по категориям и сходство нормализованных названий, а также формирует CSV.

Сравнение описывает структуру текущего информационного источника. Оно не нормируется на население, число детей, бюджет или прожиточный минимум и не измеряет объём финансирования, доступность услуг, щедрость либо эффективность семейной политики.

## Быстрый локальный запуск

Требуется Node.js 22 или новее.

```bash
npm ci
npm run check
npm test
npx playwright install --with-deps chromium
npm run test:e2e
npm run serve
```

После запуска страницы доступны по адресам:

```text
http://localhost:8080/index.html
http://localhost:8080/situations.html
http://localhost:8080/compare.html
http://localhost:8080/methodology.html
```

Открывать HTML напрямую через `file://` не следует: браузер ограничит загрузку локальных JSON-файлов.

## Проверки

`npm run check` проверяет обязательные страницы и локальные ресурсы, уникальность идентификаторов, наличие методологических оговорок, число и структуру регионов, соответствие метаданных каталогу, полный набор подробных карточек и допустимость точных официальных ссылок.

`npm test` запускает существующие и новые модульные тесты. Новые тесты проверяют нормализацию, совместимость хеширования шардов, территориальный охват, глубокие ссылки, объяснимое ранжирование, исключение карточек других регионов, раздельный учёт федерального и регионального уровней, сходство названий и экспорт CSV.

`npm run test:e2e` запускает существующую Playwright-конфигурацию для desktop и mobile. Дополнительный сценарий проверяет каталог, избранное, устойчивый URL карточки, страницу подбора, отсутствие чувствительных ответов в URL, сравнение двух регионов, методологию и горизонтальное переполнение.

## Обновление каталога

Патч не меняет действующий процесс обновления. Рекомендуемый ручной прогон:

```bash
npm ci
npx playwright install --with-deps chromium
npm test
npm run update
npm run check
npm run test:e2e
npm run serve
```

Новый снимок должен публиковаться только после успешной проверки. Технически успешный парсинг не следует интерпретировать как юридическую верификацию всех сумм и критериев.

## Данные и качество

Краткие карточки загружаются из `site/data/measures.json`. Подробные условия разделены на существующие шарды `site/data/details/*.json` и загружаются по требованию. Страница методологии динамически показывает дату снимка, число записей, территориальное покрытие и число точных официальных ссылок из метаданных.

При отсутствии точной официальной ссылки карточка явно предупреждает о необходимости сверки в ведомстве или МФЦ. Автоматическое присвоение юридического уровня доказательности не выполняется, поскольку текущая схема не содержит нормативных оснований для всех карточек.

Проектная схема следующего поколения описана в `docs/data-model-v3.md`. Она разделяет территорию действия, поставщика, правовой статус, тип помощи, критерии семьи, доход, имущество, занятость, сроки, каналы обращения, период действия и доказательства.

## Приватность

Платформа не требует аккаунтов. Анкета подбора работает локально. Избранное хранится в `localStorage` браузера и не синхронизируется между устройствами. При внедрении продуктовой аналитики не следует передавать свободный текст, чувствительные признаки анкеты или рекламные идентификаторы. Рекомендуемые события и показатели описаны в `docs/product-metrics.md`.

## Исправления

Подробная карточка формирует ссылку на создание issue в репозитории с идентификатором, названием и регионом меры. Пользователю предлагается не указывать персональные данные и приложить официальный источник. Редакционная процедура описана в `docs/editorial-policy.md`.

## Лицензирование

Патч не выбирает лицензию за владельца проекта. Перед объявлением кода и данных открытыми необходимо отдельно определить права на программный код, собственные тексты, структурированный массив, материалы информационного партнёра и логотипы. Перечень решений находится в `docs/licensing-todo.md`.

## Применение поставляемого патча

Распакуйте архив и выполните из его корня:

```bash
node apply-platform-patch.mjs /абсолютный/путь/к/family-support
```

Установщик проверяет структуру целевого репозитория, создаёт резервную копию заменяемых файлов и копирует только файлы интерфейса, тестов, проверки и документации. `site/data/`, парсер, workflows, assets и vendor не перезаписываются.

После применения:

```bash
cd /абсолютный/путь/к/family-support
npm run check
npm test
npm run test:e2e
```

Подробный состав изменений приведён в `PATCH-NOTES.md` и `APPLY.md`.
