# Управление корпусом стратегий

## Статус массива

Корпус сформирован из файлов, переданных владельцем проекта. Техническая обработка включает унификацию форматов, извлечение метаданных, расчёт контрольных сумм и лексических показателей. Она не равнозначна правовой верификации последней редакции.

## Минимальный редакционный цикл

Для каждого документа рекомендуется хранить официальный URL публикации, реквизиты акта, дату принятия, сведения о редакции, период действия, дату последней ручной проверки и имя ответственного редактора. Изменение PDF должно сопровождаться обновлением SHA-256, числа страниц, размера, манифеста и лексического профиля.

Новая версия не должна бесследно заменять прежнюю. Для исследовательской воспроизводимости предпочтительно сохранять архив редакций и поле `supersedes` либо отдельный журнал изменений.

## Уровни проверки

| Уровень | Требование |
|---|---|
| Технический | Файл открывается, имеет ожидаемый формат, размер и контрольную сумму |
| Библиографический | Установлены название, территория, вид документа, период и реквизиты |
| Официальный | Найдена публикация на официальном ресурсе и подтверждена редакция |
| Содержательный | Эксперт проверил полноту, цели, показатели и структуру мероприятий |

Интерфейс текущей версии гарантирует только техническую проверку и частично библиографическое описание. Остальные уровни должны фиксироваться отдельно.

## Лицензии и права

До публичного распространения необходимо определить правовое основание размещения каждого документа и исходного файла, особенно документов, полученных не с официальных сайтов. Следует разделять лицензию программного кода, условия повторного использования структурированных метаданных и режим публикации текстов документов.

## Добавление документа

Новый документ помещается в соответствующую директорию `site/documents/strategies/`. Затем обновляются `site/data/strategies.json` и CSV-манифест, после чего запускается сценарий лексического профилирования и полный контроль:

```bash
python3 scripts/profile-strategy-texts.py
npm run check:comparison
npm run test:comparison
npm run test:e2e
```

Перед коммитом следует проверить, что ни один файл не превышает 100 MiB и что суммарный размер опубликованного сайта остаётся в пределах ограничений выбранной инфраструктуры.
